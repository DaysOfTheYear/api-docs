## Overview

Retrieve detailed information for specific events by providing their IDs or slugs. This endpoint returns full event data including content, metadata, and related information.

<Warning>
This endpoint is **only available** to Internal tier users.
</Warning>

## Endpoint

```
GET /api/v2/events/
```

## Query Parameters

<ParamField body="identifiers" type="array" required={true}>
  Array of event identifiers. Can be either:
  - **Numeric IDs**: Post IDs (e.g., `12345`)
  - **Slugs**: URL-friendly names (e.g., `"coffee-day"`)
  
  The endpoint auto-detects the type: numeric values are treated as IDs, non-numeric values as slugs.
  
  Example: `["12345", "coffee-day", "tea-day"]`
</ParamField>

<ParamField body="timezone_offset" type="integer" required={false}>
  Timezone offset from UTC. Must be between -12 and 12. Defaults to 0 (UTC).
</ParamField>

## Request Examples

### Get events by IDs

```bash
curl -H 'X-Api-Key: YOUR_API_KEY' \
  'https://www.daysoftheyear.com/api/v2/events/?identifiers[]=12345&identifiers[]=67890'
```

### Get events by slugs

```bash
curl -H 'X-Api-Key: YOUR_API_KEY' \
  'https://www.daysoftheyear.com/api/v2/events/?identifiers[]=coffee-day&identifiers[]=tea-day'
```

### Mixed IDs and slugs

```bash
curl -H 'X-Api-Key: YOUR_API_KEY' \
  'https://www.daysoftheyear.com/api/v2/events/?identifiers[]=12345&identifiers[]=tea-day'
```

### Using JSON (POST request)

```bash
curl -X POST \
  -H 'X-Api-Key: YOUR_API_KEY' \
  -H 'Content-Type: application/json' \
  -d '{"identifiers": ["12345", "coffee-day"]}' \
  'https://www.daysoftheyear.com/api/v2/events/'
```

### Response Fields

<ResponseField name="id" type="integer">
  Unique identifier for the event
</ResponseField>

<ResponseField name="name" type="string">
  Name of the event
</ResponseField>

<ResponseField name="url" type="string">
  Full URL to the event page on Days Of The Year
</ResponseField>

<ResponseField name="excerpt" type="string">
  First 50 words of content or bespoke summary
</ResponseField>

<ResponseField name="content" type="string">
  Full article content in HTML format
</ResponseField>

<ResponseField name="type" type="string">
  Type of event: `"day"`, `"month"`, or `"other"`
</ResponseField>

<ResponseField name="next_date" type="string">
  Next occurrence date in `YYYY-MM-DD` format
</ResponseField>

<ResponseField name="grade" type="integer">
  Editorial grade (1-5) indicating content quality
</ResponseField>

<ResponseField name="cta" type="string">
  Call-to-action text
</ResponseField>

<ResponseField name="wordcount" type="integer">
  Total word count of the article
</ResponseField>

<ResponseField name="naics" type="array">
  Array of NAICS (North American Industry Classification System) codes and names
</ResponseField>

<ResponseField name="tags" type="array">
  Array of tag strings associated with the event
</ResponseField>

<ResponseField name="hashtags" type="array">
  Array of hashtag strings (without the # symbol)
</ResponseField>

<ResponseField name="countries" type="array">
  Array of country codes (ISO 3166-1 alpha-2) where the event is observed
</ResponseField>

<ResponseField name="meta" type="object">
  Additional metadata object (varies by event)
</ResponseField>

<ResponseField name="related" type="array">
  Array of related event objects with `id`, `name`, and `url`
</ResponseField>

<ResponseField name="dates" type="array">
  Array of date objects with `date` and `type` fields
</ResponseField>

<ResponseField name="date_type" type="string">
  Type of date pattern: `"fixed"`, `"floating"`, or `"recurring"`
</ResponseField>

<ResponseField name="date_pattern" type="string">
  Pattern describing how the date is calculated
</ResponseField>

<ResponseField name="status" type="string" internal>
  Post status (`"publish"` or `"draft"`). Only included for Internal tier users.
</ResponseField>

<ResponseField name="meta_description" type="string" internal>
  SEO meta description. Only included for Internal tier users.
</ResponseField>

<ResponseField name="has_faq" type="boolean" internal>
  Whether the post contains FAQ blocks. Only included for Internal tier users.
</ResponseField>

<ResponseField name="has_facts" type="boolean" internal>
  Whether the post contains facts blocks. Only included for Internal tier users.
</ResponseField>

<ResponseField name="has_timeline" type="boolean" internal>
  Whether the post contains timeline blocks. Only included for Internal tier users.
</ResponseField>

<ResponseField name="agentic_metadata" type="object" internal>
  Agentic research metadata. Only included for Internal tier users.
</ResponseField>

## Internal Tier Additional Fields

Internal tier users receive additional fields not available to other tiers:

- `status`: Post publication status
- `meta_description`: SEO meta description
- `has_faq`, `has_facts`, `has_timeline`: Block presence indicators
- `agentic_metadata`: Research and processing metadata
- Extended `meta` object with grading information

## Error Responses

#### Missing identifiers

```json
{
  "code": 400,
  "message": "identifiers parameter is required",
  "data": [],
  "meta": {
    "request_type": "EVENTS"
  }
}
```

#### Invalid identifiers format

```json
{
  "code": 400,
  "message": "identifiers must be an array",
  "data": [],

  }
}
```

### 401 Unauthorized

Missing or invalid API key:

```json
{
  "code": 401,
  "message": "Authentication required"
}
```

### 403 Forbidden

Endpoint not available for your tier:

```json
{
  "code": 403,
  "message": "Sorry, you are not allowed to access this endpoint."
}
```

## Notes

- Results maintain the order of identifiers provided in the request
- Invalid identifiers (non-existent IDs or slugs) are silently skipped
- If no valid identifiers are found, an empty `data` array is returned with `response_count: 0`
- Internal tier users can access both published and draft posts
- The endpoint auto-detects whether identifiers are IDs or slugs based on whether they're numeric
