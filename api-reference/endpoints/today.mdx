## Overview

Returns events for today's date based on the server's timezone (or a specified timezone offset).

<Info>
This endpoint is available to all API tiers: Free, Pro, Enterprise, and Internal.
</Info>

## Endpoint

```
GET /api/v2/today/
```

## Authentication

Required. Include your API key in the `X-Api-Key` header.

## Query Parameters

<ParamField body="limit" type="integer" required={false}>
  Limit the number of results returned. If not specified, all matching events are returned.
</ParamField>

<ParamField body="timezone_offset" type="integer" required={false}>
  Timezone offset from UTC. Must be between -12 and 12. Defaults to 0 (UTC).
  
  Example: `-5` for EST, `+1` for CET
</ParamField>

<Warning>
Filtering parameters are only available for **Pro** and **Enterprise** tiers. Free tier requests with filter parameters will be ignored (filters not applied).
</Warning>

<Info>
For comprehensive filtering documentation, see the [Filtering Guide](/api-reference/filtering).
</Info>

<ParamField body="tags[]" type="array" required={false}>
  Filter events by tags. Can specify multiple tags. Matches by tag name or slug.
  
  Example: `tags[]=food&tags[]=beverage`
</ParamField>

<ParamField body="naics[]" type="array" required={false}>
  Filter events by NAICS industry codes. Can specify multiple codes.
  
  Example: `naics[]=311920&naics[]=722511`
</ParamField>

<ParamField body="grade_min" type="integer" required={false}>
  Filter events by minimum editorial grade. Must be between 1 and 5.
  
  Example: `grade_min=4` returns only events with grade 4 or 5
</ParamField>

<ParamField body="date_type" type="string" required={false}>
  Filter events by date type. Must be one of: `"fixed"`, `"floating"`, or `"recurring"`.
  
  Example: `date_type=fixed`
</ParamField>

<ParamField body="countries[]" type="array" required={false}>
  Filter events by country codes (ISO 3166-1 alpha-2). Can specify multiple countries.
  
  Example: `countries[]=US&countries[]=CA`
</ParamField>

<ParamField body="event_type" type="string" required={false}>
  Filter events by event type. Must be one of: `"day"`, `"month"`, or `"other"`.
  
  Example: `event_type=day`
</ParamField>

<ParamField body="filter_operator" type="string" required={false}>
  Operator for combining multiple filter values. Must be `"and"` or `"or"`. Defaults to `"and"`.
  
  - `"and"`: Event must match ALL specified filter values (e.g., all tags, all NAICS codes)
  - `"or"`: Event must match ANY of the specified filter values (e.g., any tag, any NAICS code)
  
  Example: `filter_operator=or`
</ParamField>

## Request Examples

### Basic Request

```bash
curl -H 'X-Api-Key: YOUR_API_KEY' \
  'https://www.daysoftheyear.com/api/v2/today/?limit=10&timezone_offset=-5'
```

### With Filters (Pro/Enterprise Only)

Filter events by tags and NAICS codes using AND logic:

```bash
curl -H 'X-Api-Key: YOUR_API_KEY' \
  'https://www.daysoftheyear.com/api/v2/today/?tags[]=food&tags[]=beverage&naics[]=311920&grade_min=4&filter_operator=and'
```

Filter events using OR logic (event must have any of the specified tags or NAICS codes):

```bash
curl -H 'X-Api-Key: YOUR_API_KEY' \
  'https://www.daysoftheyear.com/api/v2/today/?tags[]=food&tags[]=beverage&naics[]=311920&filter_operator=or'
```

Filter by country and minimum grade:

```bash
curl -H 'X-Api-Key: YOUR_API_KEY' \
  'https://www.daysoftheyear.com/api/v2/today/?countries[]=US&countries[]=CA&grade_min=4&filter_operator=and'
```

### Response Fields

<ResponseField name="id" type="integer">
  Unique identifier for the event
</ResponseField>

<ResponseField name="name" type="string">
  Name of the event
</ResponseField>

<ResponseField name="url" type="string">
  Full URL to the event page on Days Of The Year
</ResponseField>

<ResponseField name="excerpt" type="string">
  First 50 words of content or bespoke summary
</ResponseField>

<ResponseField name="type" type="string">
  Type of event: `"day"`, `"month"`, or `"other"`
</ResponseField>

<ResponseField name="next_date" type="string">

</ResponseField>

<ResponseField name="grade" type="integer">
  Editorial grade (1-5) indicating content quality
</ResponseField>

<ResponseField name="tags" type="array">
  Array of tag strings associated with the event
</ResponseField>

<ResponseField name="countries" type="array">
  Array of country codes (ISO 3166-1 alpha-2) where the event is observed
</ResponseField>

<ResponseField name="date_type" type="string">
  Type of date pattern: `"fixed"`, `"floating"`, or `"recurring"`
</ResponseField>

<ResponseField name="date_pattern" type="string">
  Pattern describing how the date is calculated
</ResponseField>

## Field Filtering by Tier

### Free Tier

Free tier users have the following fields **removed** from responses:
- `naics` (NAICS industry codes)
- `content` (full article content)

All other fields are included.

### Pro, Enterprise, and Internal Tiers

All fields are included in the response.

## Error Responses

Missing or invalid API key:

```json
{
  "code": 401,
  "message": "Authentication required"
}
```

### 403 Forbidden

Endpoint not available for your tier (should not occur for `/today`):

```json
{
  "code": 403,
  "message": "Sorry, you are not allowed to access this endpoint."
}
```

## Filtering

This endpoint supports advanced filtering for Pro and Enterprise tiers. See the [Filtering Guide](/api-reference/filtering) for comprehensive documentation on all available filter parameters and best practices.

### Quick Reference

- **Tags**: `tags[]=food&tags[]=beverage`
- **NAICS Codes**: `naics[]=311920`
- **Minimum Grade**: `grade_min=4`
- **Date Type**: `date_type=fixed`
- **Countries**: `countries[]=US&countries[]=CA`
- **Event Type**: `event_type=day`
- **Filter Operator**: `filter_operator=and` (default) or `filter_operator=or`

## Notes

- Results are sorted by editorial grade (highest first), then by type (day > month > other)
- The `cache_status` field indicates whether the response was served from cache
- Free tier users must include attribution when displaying content (Pro and Enterprise tiers do not require attribution)
- Timezone offset affects which date is considered "today"
- Filtering is only available for Pro and Enterprise tiers
